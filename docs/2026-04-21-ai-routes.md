# AI Routes ‚Äî 2026-04-21

## What Was Added

Two new backend API routes that use the Gemini API to analyze food and return structured nutrition data. No frontend changes were made in this step.

| Route | Method | Input | Output |
|---|---|---|---|
| `/api/ai/text` | POST | `{ "query": "string" }` | `AIResponse` JSON |
| `/api/ai/image` | POST | `multipart/form-data` with `image` field | `AIResponse` JSON |

Both return the same `AIResponse` shape:

```typescript
{
  description: string;        // what Gemini identified
  items: AIFoodItem[];        // per-component breakdown
  totalCalories: number;      // sum across all items √ó quantity
  totalProtein: number;
  totalCarbs: number;
  totalFat: number;
}
```

## Files Added

- `lib/gemini.ts` ‚Äî shared Gemini client setup, prompt, JSON extractor, response validator
- `types/ai.ts` ‚Äî `AIFoodItem` and `AIResponse` types
- `app/api/ai/text/route.ts` ‚Äî text query handler
- `app/api/ai/image/route.ts` ‚Äî image upload handler

## Technical Decisions & Rationale

### Model: `gemini-2.5-flash-lite`
Chosen by the user. Fast, cost-effective, and multimodal (supports both text and image inputs). The same model handles both routes ‚Äî no reason to use different models for text vs. image.

### SDK: `@google/generative-ai`
The established Google AI SDK for JavaScript. Supports Gemini 2.x models, works server-side in Next.js API routes. `inlineData` is used to pass image bytes directly ‚Äî no need for Google Cloud Storage or signed URLs.

### Prompt strategy: structured JSON instruction
The prompt instructs Gemini to return **only valid JSON** with no code fences or explanation. In practice, Gemini occasionally wraps the response in `\`\`\`json ... \`\`\`` code fences despite the instruction. `lib/gemini.ts` handles this with `extractJSON()` ‚Äî it strips fences first, then falls back to finding the first `{...}` block in the response. This makes parsing resilient to minor format deviations.

### Response normalization in `buildResponse()`
Rather than trusting Gemini's exact types, all fields are coerced: `Math.round(Number(...))` for integers, `Math.round(Number(...) * 10) / 10` for 1-decimal floats, `String(...)` for strings. This prevents type errors if Gemini returns a calorie as `"350"` instead of `350`.

### Total fields computed server-side
`totalCalories`, `totalProtein`, `totalCarbs`, `totalFat` are summed from `items` on the server rather than asking Gemini to compute them. This ensures the totals are always consistent with the items array regardless of any rounding Gemini does.

### Image validation
- Accepted MIME types: `image/jpeg`, `image/png`, `image/webp`, `image/heic`, `image/heif`
- Max size: 10 MB (Gemini's inline data limit is 20 MB; 10 MB is conservative)
- Type and size checked before the Gemini call to fail fast with a clear error

### GEMINI_API_KEY never exposed to the browser
Both routes are server-side (`app/api/`). The key lives in `.env` and is read only via `process.env.GEMINI_API_KEY` inside the route handler. It is never passed to client components and never prefixed with `NEXT_PUBLIC_`.
